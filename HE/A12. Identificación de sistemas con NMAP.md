## Introducción

Este documento detalla los resultados obtenidos durante la fase de reconocimiento activo de una auditoría de tipo caja negra. Se ha utilizado la herramienta **Nmap** para identificar la superficie de ataque de un servidor corporativo, analizando servicios, versiones y configuraciones críticas.

El objetivo principal es proporcionar una visión completa de las vulnerabilidades y debilidades de seguridad presentes en el sistema 192.168.56.101, permitiendo diseñar una estrategia de mitigación efectiva.

---

## Fase de Recopilación de Información

### a) Identificación de la IP del equipo objetivo

Se realizó un barrido ICMP (Ping Sweep) en el segmento de red para localizar hosts activos.

**Comando utilizado:**

```bash
sudo nmap -sn 192.168.56.0/24
```

**Resultado:** El equipo objetivo se encuentra en la dirección IP **192.168.56.101**.

<img width="785" height="567" alt="image" src="https://github.com/user-attachments/assets/474d5af3-9c5f-44d2-b95f-ead257cf7cf3" />

### b) Identificación del Sistema Operativo

Utilizando el análisis de la pila TCP/IP, se determinó la firma del sistema operativo mediante análisis pasivo y activo.

**Comando utilizado:**

```bash
sudo nmap -O 192.168.56.101
```

**Nota técnica:** El sistema devuelve huellas compatibles con un **Kernel Linux (versiones 2.6.X a 3.X)**. La presencia de servicios como Samba y Apache confirma una distribución basada en **Ubuntu (Metasploitable 3)**.

### c) Identificación de puertos y servicios (TCP / UDP)

Se ejecutó un escaneo exhaustivo para descubrir puertas de enlace de comunicación y servicios activos.

**Escaneo TCP (Todos los puertos):**

```bash
nmap -p- --open --min-rate 5000 192.168.56.101
```

```bash
sudo nmap -sU --top-ports 20 192.168.56.101
```

### d) Análisis de versiones de servicios

Para determinar la superficie de vulnerabilidad, se interrogaron los banners de los servicios detectados mediante scripts de detección de versiones.

**Comando utilizado:**

```bash
nmap -p 21,22,80,111,139,445,3306,8080 --min-rate 5000 -sV 192.168.56.101
```

### e) Verificación de credenciales y usuarios (NSE)

Se utilizaron scripts de enumeración del Nmap Scripting Engine para detectar debilidades en la autenticación y posibilidad de acceso con credenciales débiles.

**Comando utilizado:**

```bash
nmap --script auth -p 21,22,445,3306 192.168.56.101
```

### f) Análisis de Vulnerabilidades críticas (NSE)

Se ejecutó la categoría `vuln` de Nmap Scripting Engine para cruzar versiones con bases de datos de vulnerabilidades conocidas (CVE).

**Resultado de la ejecución:**

#### Vulnerabilidades Detectadas:

**Vulnerabilidades Web:**
- Se detectaron múltiples puntos de **SQL Injection** en la raíz del servidor y en `/phpmyadmin/`
- Se encontraron múltiples puntos de **Cross-Site Scripting (XSS)** basados en DOM en `/phpmyadmin/js/functions.js`
- Posibles vulnerabilidades **CSRF** en `/payroll_app.php`, `/drupal/`, y `/chat/`
- Directorios con listado habilitado que facilitan la enumeración

**Denegación de Servicio (DoS):**
- Vulnerabilidad **Slowloris (CVE-2007-6750)** que permite a un atacante dejar fuera de servicio las aplicaciones web corporativas

**SMB:**
- Vulnerabilidades potenciales en la gestión de registros remotos (smb-vuln-regsvc-dos)

## Informe Técnico Final

### Resumen Ejecutivo

El sistema **192.168.56.101** presenta una **postura de seguridad CRÍTICA** con múltiples vulnerabilidades explotables sin autenticación:

**Vectores de Ataque Principales:**
- **Software desactualizado** (ProFTPD 1.3.5, Apache 2.4.7, MySQL) → RCE
- **SQL Injection** en raíz y `/phpmyadmin/` → Extracción de datos
- **XSS y CSRF** en aplicaciones web
- **Slowloris (CVE-2007-6750)** → DoS
- **SMB con acceso nulo** → Enumeración de usuarios

### Recomendaciones Críticas

1. **Actualizar inmediatamente:** Kernel Linux 5.x+, Apache 2.4.52+, MySQL 8.0+
2. **Hardening SMB:** Deshabilitar invitados, forzar firmado de mensajes
3. **Seguridad Web:** Validación de entrada, WAF, deshabilitar listado de directorios
4. **Firewalling:** Cerrar puertos innecesarios (111, 8080, etc.)
5. **Monitoreo:** Implementar IDS/IPS y logging centralizado

### Conclusión

**Crítico:** Sistema no apto para producción sin implementar las medidas de mitigación. Se recomienda análisis de penetración post-remediación.
